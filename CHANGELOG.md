## [3.0.0] - 2026-03-04

### 新增

#### 核心功能
- ✨ **智能意图识别**: 使用 LLM 识别自然语言操作意图
- ✨ **业务操作模板**: 内置停车场相关业务操作
- ✨ **操作预览**: 变更操作前预览影响范围
- ✨ **参数验证**: 自动验证参数格式和类型
- ✨ **事务支持**: 支持多步骤事务和回滚

#### 监控告警
- 📈 **指标收集**: 实时收集操作指标（成功率、执行时间等）
- 🚨 **告警系统**: 支持错误率、执行时间阈值告警
- 📊 **结构化日志**: JSON 格式的结构化日志，便于分析
- 📧 **多渠道通知**: 支持日志、邮件、Webhook 告警

#### 安全增强
- 🔒 **SQL 注入防护**: 使用参数化查询防止 SQL 注入
- 🔒 **SQL 安全检查**: 检测危险操作（DROP、TRUNCATE 等）
- 🔒 **操作预览确认**: 变更操作前需要用户确认

#### 用户体验
- 🎨 **友好交互**: 清晰的命令行交互界面
- 💡 **智能提示**: 自动提示缺失参数和可选值
- 📋 **帮助系统**: 完整的命令和操作帮助
- 🚀 **快速操作**: 支持业务语言快速操作

### 改进

#### 性能优化
- ⚡ **连接池**: 使用 SQLAlchemy 连接池提升性能
- ⚡ **缓存机制**: 知识库和数据库结构缓存
- ⚡ **批量操作**: 支持批量下发到多个场库

#### 代码质量
- 🧪 **测试覆盖**: 单元测试覆盖率达到 80%+
- 📝 **文档完善**: 完整的用户手册、API 文档、部署文档
- 🔧 **代码重构**: 模块化设计，职责清晰

### 修复

#### Bug 修复
- 🐛 **修复**: DatabaseManager 缺少 execute_update 方法
- 🐛 **修复**: SQL 预览生成逻辑错误
- 🐛 **修复**: 字符串拼接导致的 SQL 注入风险
- 🐛 **修复**: 意图识别参数提取不准确
- 🐛 **修复**: 日志配置未正确应用

### 变更

#### 破坏性变更
- ⚠️ **配置文件**: `.env` 配置项有所变化，请参考 `.env.example`
- ⚠️ **Python 版本**: 最低 Python 版本要求提升到 3.8

#### 依赖更新
- 📦 **新增依赖**: sqlalchemy, pyyaml, pytest, pytest-cov
- 📦 **更新依赖**: pandas, openpyxl

## 版本说明

### 版本号格式

`主版本号.次版本号.修订号`

- **主版本号**: 不兼容的 API 修改
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 变更类型

- **新增 (Added)**: 新功能
- **改进 (Changed)**: 功能变更
- **废弃 (Deprecated)**: 即将移除的功能
- **移除 (Removed)**: 已移除的功能
- **修复 (Fixed)**: Bug 修复
- **安全 (Security)**: 安全相关的修复

## 如何升级

### 从 2.x 升级到 3.0.0

1. **备份配置**
   ```bash
   cp .env .env.backup
   ```

2. **更新依赖**
   ```bash
   pip install --upgrade -r requirements.txt
   ```

3. **更新配置文件**
   参考 `.env.example` 更新 `.env`

4. **测试功能**
   ```bash
   python main.py
   ```

### 从 1.x 升级到 2.0.0

1. **备份配置**
   ```bash
   cp .env .env.backup
   ```

2. **更新依赖**
   ```bash
   pip install --upgrade pandas openpyxl requests
   ```

3. **测试功能**
   ```bash
   python main.py
   ```
