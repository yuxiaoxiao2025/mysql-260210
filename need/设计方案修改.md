2. 相似度分数误导（数据级）
位置：6.2节 `_search_tables` 和 `_search_fields`

问题：ChromaDB返回的`distances`是L2距离（越小越相似），但`TableMatch.similarity_score`命名暗示越大越好（0-1）。

修正建议（6.2节）：

```python
# 方案A：转换余弦相似度（如果是余弦距离）
similarity_score = 1 - results['distances'][0][i]  # 简化的归一化

```

3. 表级 Schema 文本过长（性能级）
位置：3.2节 Layer 1

问题：`包含字段：{columns}`会列出所有字段（可能50+个），导致：
- Token超限（text-embedding-v4限制1024 tokens）
- 关键业务字段被通用字段（`id`, `created_at`）稀释

修正建议（3.2节）：

```python
# 只取关键业务字段（过滤通用字段）
GENERIC_FIELDS = {'id', 'created_at', 'updated_at', 'is_deleted'}

key_columns = [c for c in columns if c['name'] not in GENERIC_FIELDS][:10]  # 限制前10个

schema_text = f"表名：{table_name}。描述：{comment}。关键字段：{key_columns}..."
```

4. 缺失关系图谱扩展（功能级）
位置：6.2节 `RetrievalAgent`

问题：当前仅做向量检索，没有利用外键关系扩展（如检索到`orders`后，应自动加入`users`表因为存在外键关联）。

修正建议（6.2节添加方法）：

```python
def _expand_by_foreign_keys(self, table_names: List[str]) -> List[str]:
    """基于外键关系扩展相关表（Graph Expansion）"""
    related = set(table_names)
    for table in table_names:
        # 从table_graph.json加载关系
        fks = self.graph.get_foreign_keys_from(table)
        for fk in fks:
            related.add(fk.referenced_table)
    return list(related)
```
