# 停车数据库管理工具智能化改造设计方案

## 一、改造目标

将系统从「固定流程模式」改造成「自然语言驱动 + 专属业务记忆库 + 不懂就问增量学习 + 越用越聪明」的模式。

### 核心转变

| 原模式 | 新模式 |
|-------|-------|
| 预设固定流程 | 对话式动态理解 |
| 用户适应系统 | 系统适应用户 |
| 每次重新理解 | 记住业务知识 |
| 技术导向 | 自然语言对话 |

---

## 二、系统启动流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 检测向量库增量同步（表结构变化检测）                      │
│     - 有变化 → 自动重新嵌入变更的表                          │
│     - 无变化 → 继续                                         │
├─────────────────────────────────────────────────────────────┤
│  2. 检查知识库状态                                           │
│     - 空知识库 → 启动「初始化向导」                          │
│     - 已有知识库 → 提示「是否补充知识库？」                   │
├─────────────────────────────────────────────────────────────┤
│  3. 初始化向导 / 补充知识库（每次10个问题）                   │
│     - 根据已向量化的9000+表生成问题                          │
│     - 用户回答 → 写入知识库                                  │
├─────────────────────────────────────────────────────────────┤
│  4. 进入工作模式                                             │
│     - 显示：你好，我是你的停车数据库助手，有什么可以帮你？     │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 增量同步机制

启动时自动检测数据库表结构变化：
- 比对每个表的版本哈希值
- 发现变化的表自动重新嵌入
- 更新向量库元数据

### 3.2 初始化向导

首次使用时，通过对话建立业务知识库：

**开场问题：**
1. 你一般用这个工具查询哪些内容？
2. 你会修改数据库的哪些内容？

**根据9000+表生成的示例问题：**
- "我看到数据库里有 parkcloud.cloud_park 表，你的业务里叫它什么？"
- "内部车辆和线上固定车是同一个意思吗？"
- "你说'停过某个园区'，是指进出场记录还是绑定关系？"

## 三、工作模式流程

```
用户提问
    ↓
检查上下文记忆（最近车牌、核心需求等）
    ↓
检查知识库是否有相关概念映射
    ↓
┌─────────────────┬─────────────────┐
│ 找到概念        │ 没找到概念      │
├─────────────────┼─────────────────┤
│ 用自然语言说明  │ 主动提问：      │
│ 执行意图，      │ "XX是什么意思？"│
│ 让用户确认      │ 优先给选项      │
└─────────────────┴─────────────────┘
    ↓
用户确认/回答
    ↓
新的概念映射写入知识库
    ↓
执行查询/操作
    ↓
结果显示：前10条预览 + 自动导出Excel
```

### 3.3 意图确认示例

```
用户：查一下沪BAB1565在26年停过哪些园区

系统：（查知识库，找到"园区=场库"、"停过=进出场记录"）
      我准备查询沪BAB1565在2026年1月1日到今天在所有园区的进出场记录，
      如果找到结果就发给你，可以吗？

用户：可以

系统：好的，正在查询...
```

### 3.4 不懂就问示例

```
用户：沪BAB1565最近被投诉过吗

系统：（查知识库，没有"投诉"的概念）
      "投诉"在数据库里对应什么？我没找到相关记录。
      你说的"投诉"是指：
      A. 客户投诉记录
      B. 违规记录
      C. 其他（请说明）

用户：B

系统：好的，我记住了"投诉 = 违规记录"。
      那违规记录在哪个表里查？

用户：不知道，你帮我找找

系统：我在数据库里搜索了一下，发现这些表可能相关：
      A. parkcloud.cloud_complaint（投诉记录表）
      B. p_xxx.violation_log（违规日志表）
      C. 其他（请说明表名）
      ...
```

## 四、上下文记忆系统

### 4.1 记忆规则

- **保留最近 100 轮对话**完整内容
- **记住最近一次车牌**："这辆车"指最近提到的车牌
- **记住核心信息**：核心需求、已做操作、纠正过的错误
- **每次回答前必须查看记忆内容**

### 4.2 记忆内容示例

```
当前对话记忆：
- 最近车牌：沪BAB1565
- 核心需求：查询进出场记录
- 已做操作：查询了绑定园区列表
- 纠正记录：用户说"停过"不是"绑定"，是"进出场记录"
```

## 五、业务知识库

### 5.1 概念映射结构

```json
{
  "concept_id": "parking_lot",
  "user_terms": ["园区", "场库", "停车场"],
  "database_mapping": {
    "table": "parkcloud.cloud_park",
    "description": "33个园区的信息表"
  },
  "learned_from": "初始化向导",
  "created_at": "2026-03-10",
  "confirmed_count": 5
}
```

### 5.2 预置概念映射（从原模板转换）

| 用户说法 | 数据库对应 | 说明 |
|---------|-----------|------|
| 内部车辆 | 线上固定车 | cloud_fixed_plate 表 |
| 园区/场库 | parkcloud.cloud_park | 33个园区信息 |
| 停过/去过 | 进出场记录 | p开头库的进出记录表 |
| 下发 | 设置 state=0 | 状态变更操作 |
| 绑定 | 场库关联 | cloud_fixed_plate_park 表 |

### 5.3 知识库管理

- **自动管理**：系统自动写入新学习的概念
- **用户可操作**：随时查看、编辑、删除条目
- **置信度追踪**：记录每个概念被确认的次数

## 六、交互规则

### 6.1 必须做到

- ✅ 先查知识库，查到也要用自然语言说明意图让用户确认
- ✅ 查不到就问，优先给选项，没有选项直说没听懂
- ✅ 用自然语言说明执行意图，不说技术术语
- ✅ 出错先自动尝试修复，修不好再告诉用户
- ✅ 显示前10条预览，自动导出Excel

### 6.2 绝对禁止

- ❌ 自作主张直接执行
- ❌ 听不懂瞎猜结果
- ❌ 记不住之前说过的话
- ❌ 说用户听不懂的技术术语

## 七、技术修复

### 7.1 Rerank 预算调整

- **原设置**：500ms 总预算
- **新设置**：1000ms 总预算
- **目的**：确保两层 Rerank（表级+字段级）都能执行

### 7.2 增量同步机制

```
启动时：
1. 计算每个表的版本哈希（表结构+字段注释）
2. 与上次记录的哈希比对
3. 发现变化 → 自动重新嵌入该表
4. 更新向量库和知识图谱
```

---

## 八、原有模板处理

**方案：转换成知识库初始数据**

将 `business_knowledge.yaml` 中的固定操作模板转换为概念映射，预填到新知识库中：
- 车牌查询 → plate_query 概念
- 下发操作 → plate_distribute 概念
- 绑定查询 → plate_park_bindings 概念

这样保留了已有的业务逻辑定义，同时让新系统能够在此基础上继续学习。

## 十、验收标准

1. **首次打开**：显示初始化向导，通过10个问题建立知识库
2. **再次打开**：提示是否补充知识库，回答"不需要"直接进入工作模式
3. **用户提问**：系统先查知识库，查到也说明意图让用户确认
4. **概念缺失**：系统主动提问，优先给选项
5. **上下文连贯**：说"这辆车"能正确识别最近车牌
6. **结果展示**：前10条预览 + 自动导出Excel
7. **错误处理**：先自动尝试修复，修不好再用通俗话告诉用户
8. **增量同步**：表结构变化时自动更新向量库
