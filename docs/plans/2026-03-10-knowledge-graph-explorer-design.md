# 2026-03-10 智能数据库图谱探索器设计方案

## 1. 概述
本项目旨在构建一个基于 **Streamlit** 的可视化交互应用，帮助用户通过自然语言探索数据库结构，确认所需数据表，并最终生成查询方案。核心解决“用户不知道数据在哪张表”和“多表关联复杂”的痛点。

## 2. 核心流程
1.  **意图解析**：用户输入自然语言 -> LLM 提取关键词 + Embedding 向量检索。
2.  **图谱构建与排序**：
    *   **Recall (召回)**：检索 Top 50 相关表。
    *   **Rerank (精排)**：基于语义匹配度、业务域权重、历史热度进行打分。
    *   **Render (渲染)**：Top N 节点高亮，直接邻居（外键关联）作为二级节点展示。
3.  **交互确认**：
    *   **单表确认**：点击节点 -> 查看字段注释 + Top 5 采样数据 -> 确认/排除。
    *   **多表组合**：选中多表 -> 自动推断外键路径 -> 预览 `JOIN` 后的组合数据。
4.  **结果输出**：
    *   生成 SQL 查询（带 LIMIT）。
    *   执行并展示结果表格（支持自然语言修正 SQL）。
    *   导出 Excel 或 Python 代码。

## 3. 详细设计

### 3.1 界面布局 (Streamlit)
*   **Sidebar (左侧)**
    *   **自然语言输入框**：支持多行输入。
    *   **候选表列表**：按 Rerank Score 降序排列，显示匹配理由。
    *   **已选表集合**：显示当前锁定的表（如 `parking_order`, `park_info`）。
    *   **历史记录**：下拉框选择最近 10 次查询，点击恢复上下文。
*   **Main Area (中间)**
    *   **交互式图谱** (`streamlit-agraph`)：
        *   节点大小/颜色反映相关性。
        *   边表示外键关系。
        *   支持点击展开邻居、右键菜单。
    *   **兜底搜索栏**：图谱下方提供“没找到？搜表名/字段”输入框，支持模糊匹配追加节点。
*   **Detail Panel (右侧/底部)**
    *   **Tab 1: 表详情**：字段列表、注释、数据采样（LIMIT 5）。
    *   **Tab 2: 组合预览**：多表 JOIN 后的数据切片，高亮关联列。
    *   **Tab 3: SQL 结果**：最终查询结果表格 + 统计信息 + 导出按钮。

### 3.2 关键算法与逻辑
*   **Rerank 策略**：
    *   `Score = 0.6 * SemanticSim + 0.3 * DomainMatch + 0.1 * HistoryHit`
    *   语义相似度：Embedding Cosine Similarity。
    *   业务域匹配：表所属 `business_domain` 与 Query 意图的匹配度。
*   **多表路径推断**：
    *   使用 `NetworkX` 的 `shortest_path` 算法，在 Schema Graph 中寻找选中表之间的最短连接路径。
    *   自动生成 `LEFT JOIN` 语句骨架。

### 3.3 异常处理与兜底
*   **无结果**：
    *   提供“模糊搜索”功能，基于 SQL `LIKE` 全库扫描表名和字段注释。
    *   提供“目录视图”，按业务域折叠展示所有表。
*   **查询纠正**：
    *   结果展示下方提供对话框：“结果不对？告诉我怎么改”。
    *   LLM 接收用户反馈，生成 `UPDATE` 后的 SQL（如增加 `WHERE` 条件）。

## 4. 技术栈
*   **Frontend**: Streamlit, streamlit-agraph (图谱), st-aggrid (表格)
*   **Backend**: Python 3.10+
*   **Data**: SQLAlchemy, Pandas, NetworkX
*   **AI**: DashScope (通义千问), Text-Embedding-v4
