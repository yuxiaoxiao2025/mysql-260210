## 1. 背景与问题

### 1.1 当前问题

| # | 问题 | 现象 | 影响 |
|---|------|------|------|
| 1 | **上下文丢失** | "这辆车"变成"沪A12345" | 对话不连贯，需要重复输入 |
| 2 | **语义失准** | 查出入记录→返回绑定记录 | 结果不符合预期 |
| 3 | **表名幻觉** | 发明了`parking_records` | SQL执行失败 |
| 4 | **无法预览** | 只能导出Excel后查看 | 反馈周期长 |
| 5 | **反馈困难** | 无法告诉系统对错 | 系统无法进化 |

### 1.2 根因分析

1. **上下文未传递到检索层**：`conversation_history`只在LLM生成时使用，向量检索没有上下文
2. **语义描述太简单**：`schema_text`只有"表名：xxx。描述：xxx"，无法区分相似概念
3. **LLM无表约束**：prompt中没有强制使用已知表
4. **CLI缺少预览**：用户必须导出Excel才能看到结果
5. **无反馈机制**：系统无法学习用户纠正

### 2.2 模块说明

#### 模块1: ContextEnhancer (上下文增强器)

**职责**：维护对话上下文，实现代词替换

**组件**：
- `SlotTracker`: 跟踪车牌、时间、场库等实体
- `QueryRewriter`: 将代词替换为实际值

**示例**：
```
Round 1: "查沪BAB1565的录入时间"
→ SlotTracker记录: {plate: "沪BAB1565"}

Round 2: "这辆车3月出入过哪些园区"
→ QueryRewriter: "这辆车" → "沪BAB1565"
→ 增强后查询: "沪BAB1565 3月出入过哪些园区"
```

#### 模块2: SemanticRetriever (语义检索器)

**职责**：基于向量相似度找到相关表

**现有基础**：已有`RetrievalAgent`和`RetrievalPipeline`

**改进点**：
- 使用增强后的查询进行检索
- 返回Top-K候选表

#### 模块3: TableConstraint (表约束器)

**职责**：防止LLM发明不存在的表

**实现**：
- 从检索结果获取候选表列表
- 在prompt中注入：`只能使用以下表：[表1, 表2, ...]`
- 验证生成的SQL只使用已知表

#### 模块4: CLIPreview (CLI结果预览)

**职责**：在CLI中显示查询结果预览

**功能**：
- 表格形式显示前5-10条记录
- 显示总记录数
- 快捷键支持：y(确认)/n(不对)/更多/导出
- 支持自然语言纠正

**示例**：
```
┌─────────────────────────────────────────────────────────────┐
│ 📊 查询结果预览 (共 26 条记录)                               │
├─────────────────────────────────────────────────────────────┤
│ 车牌      │ 场库名称       │ 绑定时间                       │
│ 沪BAB1565 │ 国际商务中心    │ 2026-01-15                     │
│ 沪BAB1565 │ 科技园         │ 2026-02-20                     │
│ ...       │ (更多 23 条)   │                                │
├─────────────────────────────────────────────────────────────┤
│ ❓ 这是你要的结果吗？ (y/n/更多/导出)                        │
└─────────────────────────────────────────────────────────────┘
```

#### 模块5: FeedbackParser (反馈解析器)

**职责**：解析用户反馈，学习改进

**支持的反馈类型**：
1. **快捷键反馈**：y(正确)/n(错误)
2. **自然语言纠正**："不对，我要的是出场记录"
3. **隐式反馈**：用户重新描述问题

**学习机制**：
- 记录查询-结果-反馈三元组
- 分析常见错误模式
- 调整检索权重

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 槽位跟踪不准确 | 查询重写错误 | 多实体类型支持，置信度过滤 |
| 表约束过严 | 限制LLM灵活性 | 提供足够多的候选表 |
| CLI预览性能 | 大数据集显示慢 | 限制预览条数，异步加载 |
| 反馈解析错误 | 学习方向错误 | 人工审核学习日志 |
